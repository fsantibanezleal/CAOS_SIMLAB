# 08 · S08 — Vehicle Routing Problem (CVRP) (use-case node)

A delivery fleet leaves a single **depot**, must serve a set of **customers** — each with an integer
**demand** — and return to the depot, with **no vehicle exceeding its capacity** $Q$, minimizing the fleet's
**total travel distance**. This is the **Capacitated Vehicle Routing Problem (CVRP)**: pure combinatorial
optimization (NP-hard), one synthetic-grid instance solved once — no time windows, no stochastics. What
makes S08 a *teaching* exhibit is that it runs the **same instance through two VRP engines** — a
**general-purpose engine (OR-Tools)** and a **state-of-the-art one (PyVRP)** — and shows the contrast:
**OR-Tools** routing is the primary plan (it minimizes total distance **plus** a global-span penalty that
*balances* the routes), while **PyVRP**'s Hybrid Genetic Search is the state-of-the-art counterweight (it
minimizes **pure total distance**, usually shorter but less balanced).
Both engines are native code, so this is a **precompute-lane** scenario — the committed trace holds both
plans and the web app replays the optimized routes as vehicles driving the network.
